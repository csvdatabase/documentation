## Create A Database In Memory

<Tabs groupId="language">
<TabItem value="ts" label="TypeScript">

```ts
import { CSDBDatabase } from "@csdb/typescript";

const db = CSDBDatabase.parse(`--- csdb
format: CSDB
version: 1
name: demo
tables:
  - notes

--- table:notes:schema
name: notes
columns:
  id: text
  body: text
required:
  - id
primary_key:
  columns: [id]

--- table:notes:data
id,body
n_001,hello
`);
```

</TabItem>
<TabItem value="py" label="Python">

```py
from csdb import CSDBDatabase

db = CSDBDatabase.parse("""--- csdb
format: CSDB
version: 1
name: demo
tables:
  - notes

--- table:notes:schema
name: notes
columns:
  id: text
  body: text
required:
  - id
primary_key:
  columns: [id]

--- table:notes:data
id,body
n_001,hello
""")
```

</TabItem>
</Tabs>

## Insert And Query Rows

<Tabs groupId="language">
<TabItem value="ts" label="TypeScript">

```ts
db.table("notes").insert({ id: "n_002", body: "from the fluent API" });
db.table("notes").insert({ id: "n_003", body: null });

const notes = db.table("notes")
  .where("body", "!=", null)
  .orderBy("id")
  .all();
```

</TabItem>
<TabItem value="py" label="Python">

```py
db.table("notes").insert({"id": "n_002", "body": "from the fluent API"})
db.table("notes").insert({"id": "n_003", "body": None})

notes = (
    db.table("notes")
    .where("body", "!=", None)
    .order_by("id")
    .all()
)
```

</TabItem>
<TabItem value="sql" label="SQL">

```sql
INSERT INTO notes (id, body)
VALUES ('n_002', 'from SQL');

INSERT INTO notes (id, body)
VALUES ('n_003', NULL);

SELECT *
FROM notes
WHERE body IS NOT NULL
ORDER BY id ASC;
```

</TabItem>
<TabItem value="json" label="JSON">

Send each object below as a separate `POST /v1/commands` request:

```json
{
  "database": "demo",
  "command": {
    "kind": "insert",
    "table": "notes",
    "rows": [{ "id": "n_002", "body": "from JSON" }]
  }
}
```

```json
{
  "database": "demo",
  "command": {
    "kind": "insert",
    "table": "notes",
    "rows": [{ "id": "n_003", "body": null }]
  }
}
```

```json
{
  "database": "demo",
  "command": {
    "kind": "select",
    "table": "notes",
    "columns": "*",
    "joins": [],
    "where": {
      "type": "is-null",
      "expr": { "type": "identifier", "name": "body" },
      "not": true
    },
    "orderBy": [{ "column": "id", "direction": "asc" }],
    "output": "objects"
  }
}
```

</TabItem>
</Tabs>

Optional non-primary-key columns accept `null`. In CSV, unquoted empty fields
represent `null`, while quoted empty strings remain empty strings.

## Save A File

<Tabs groupId="language">
<TabItem value="ts" label="TypeScript">

```ts
import { saveCSDB } from "@csdb/typescript";

await saveCSDB(db, "demo.csdb", { machineIndexes: "omit" });
```

</TabItem>
<TabItem value="py" label="Python">

```py
from csdb import save_csdb

save_csdb(db, "demo.csdb", machine_indexes="omit")
```

</TabItem>
</Tabs>

The JSON server does not accept save paths. It resolves `database` inside its
configured data directory and automatically persists successful mutations.
